# GitHub Workflow

## Branches

Use short descriptive branches:

- `feature/<name>`
- `fix/<name>`
- `docs/<name>`
- `chore/<name>`

Codex-created branches use the app default `codex/` prefix unless a human requests otherwise.

## Commits

Use conventional commits:

- `feat(merchant): add api keys`
- `fix(transaction): reject invalid status transition`
- `docs(readme): add quick start screenshots`
- `test(webhook): cover retry scheduling`
- `chore(ci): add docker build job`
