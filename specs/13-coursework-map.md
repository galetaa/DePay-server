# 13. Coursework Requirements Map

## СУБД

Используем PostgreSQL. SQLite и Access не используются.

## Схема данных

Схема должна быть получена из DBeaver по реальной БД. Стрелки разложить без сильных пересечений.

## Нормализация

Объяснить 3НФ:

- справочники отдельно;
- KYC documents отдельно;
- wallets отдельно от balances;
- invoices отдельно от transactions;
- audit/history отдельно от основных таблиц.

## Запросы

12 сложных SQL-запросов:

- JOIN;
- GROUP BY;
- COUNT/SUM/AVG;
- фильтрация по периодам;
- параметры;
- оконные функции хотя бы в одном запросе.

## Функции/процедуры

Все запросы вызываются через PostgreSQL functions. Больше 3 функций имеют параметры.

## Триггеры

Минимум 5, план — 7 разных типов:

- validation;
- history;
- status flow;
- timestamp automation;
- audit;
- risk alert.

## Интерфейс

Должен:

- показывать таблицы;
- запускать функции;
- показывать результат;
- принимать параметры;
- показывать ошибки;
- показывать графики.

## Графики

Минимум 3:

1. bar — оборот магазинов;
2. pie — статусы транзакций;
3. line — динамика платежей;
4. optional — ошибки по сетям.

## Пояснительная записка

Внутри записки оставить:

- формулировки запросов;
- SQL-код запросов;
- функции;
- процедуры;
- триггеры;
- код построения графиков;
- скриншоты результатов.
